WebSpeed SpeedStart

For WebSpeed to run successfully, both your Web Server and the WebSpeed components need to be configured properly. Visit the WebSpeed Developer's Corner to access online versions of the WebSpeed Documentation and other detailed information.

Starting WebSpeed Workshop

  1. Before processing, please make sure your Web server is running. The Web server will need to be restarted if the product was just installed and a virtual directory was created.
  2. WebSpeed Workshop consists of the AppBuilder development tool and the WebSpeed Transaction Server.
  3. Start the WebSpeed Transaction Server by using the Progress Explorer Tool (Windows), or the wtbman command line utility.
  4. To start the browser-based WebTools, you can enter the following URL (e.g. http://host-name/scripts/WebSpeed-Messenger/workshop) Or select Tools ... WebTools from AppBuilder.
  5. To access WebSpeed sample applications, see OpenEdge Getting Started: WebSpeed.

Verify WebSpeed Configuration

The following sample URL's demonstrate how to validate your WebSpeed configuration by testing the various components. Each link will return a page to validate the configuration of a WebSpeed component (e.g. Messenger) and the ability to use the configuration data to connect to the next component. Just substitute host-name, scripts and SampleSrvc with your system's values.

NOTE: Before you can run any administration utilities, you must enable internal administration commands on the WebSpeed Messenger. You can do this through Progress Explorer in the Advanced Features section of the Messenger's Properties dialog box. Select the Internal Adminstration Commands check box.

Optionally, it is possible to validate your configuration through the Messenger Administration page. This can be accessed by specifying ?WSMAdmin after the Messenger name, e.g. http://host-name/scripts/WebSpeed-Messenger?WSMAdmin

http://host-name/scripts/cgiip.exe?VerifyConfig&Level=1 CGI Messenger: Msngr-NameServer Verify
Round trip test from a Browser to the CGI Messenger (cgiip.exe). This test will retrieve configuration information from the Messenger for the Controlling NameServer. A connection will be attempted to retrieve the registered Broker list.

http://host-name/scripts/cgiip.exe?VerifyConfig&Level=2 CGI Messenger: Broker Verify
Round trip test from a Browser to the WebSpeed Broker using the CGI Messenger (cgiip.exe). This test instructs the Messenger to make a connection to the Broker and report the results.

http://host-name/scripts/cgiip.exe/ping CGI Messenger: Agent and Workshop Verify
Round trip test from a Browser to a WebSpeed Agent and WebTools using the CGI Messenger (cgiip.exe). This test instructs the Messenger to make a connection to the Broker and an available Agent. The Agent then runs a Workshop "ping" utility and reports back Agent information.
